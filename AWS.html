AWS基礎まとめ

AWSは「家を建てる」イメージで覚えると整理しやすいです。 

AWS

アベイラビリティゾーン（Availability Zone）（アベイラビリティ ゾーン）
一言でいうと データセンターのまとまり。

東京リージョンには複数のAZがあります。

例え 同じ市内に複数の営業所があるイメージ。 1つ停電しても他でサービスを継続できます。
高可用性（止まりにくいシステム）を実現するため、 EC2を複数AZへ配置することがあります。
VPC（Virtual Private Cloud）（ブイピーシー）
一言でいうと AWSの中に作る、自分専用のネットワーク。
例え 土地。 まず土地(VPC)を買って、 その中に建物を建てます。
サブネット（Subnet）（サブネット）
一言でいうと VPCを区画分けしたもの。
例え 住宅街の一区画。 土地(VPC)の中を 「住宅用」「商業用」に分けるイメージ。
よく聞かれる質問 「VPCとサブネットの違い」 → VPC＝土地 サブネット＝土地の一区画
ロードバランサー（Load Balancer）（ロードバランサー）
一言でいうと アクセスを複数サーバーへ振り分ける装置。
アクセス1000件 ↓ EC2① 500件 EC2② 500件
例え レジ係。 1人しかいないと行列になりますが、 複数レジへ案内すると待ち時間が減ります。
DNS（Domain Name System）（ディー・エヌ・エス）
一言でいうと ドメイン名をIPアドレスへ変換する仕組み。
www.amazon.co.jp ↓ 54.xx.xx.xx
例え スマホの連絡先。 「山田さん」 ↓ 電話番号
Elastic（エラスティック）
英単語の意味 Elastic＝伸縮自在・弾力がある

AWSでは 「必要に応じて増やしたり減らしたりできる」 という意味で使われています。

Elastic IP Elastic Load Balancing Elastic Beanstalk など。
Amazon DynamoDB（ダイナモディービー）
一言でいうと 高速なNoSQLデータベース。
使用例 ・オンラインゲームのランキング ・チャット ・ショッピングカート ・IoT
オンラインゲームでは プレイヤーのスコアが頻繁に更新されます。 DynamoDBは高速なので、 ランキング表示に向いています。
NoSQL（Not Only SQL）（ノーエスキューエル）
一言でいうと
リレーショナルデータベース（RDB）のような「表形式」に縛られないデータベースです。

なぜ作られた？

従来のRDBは、 ・データの整合性が高い ・複雑な検索が得意 という長所があります。 一方で、 SNSやゲームのように大量のデータを高速に読み書きする用途では、 処理が追いつかなくなることがあります。 そこで生まれたのがNoSQLです。

Amazon Route53（ルートフィフティスリー）
一言でいうと AWSのDNSサービス。
利用者 ↓ Route53 ↓ ロードバランサー ↓ EC2
53はDNSで使われるポート番号53から来ています。
レイテンシー（Latency）（レイテンシー）
一言でいうと 通信の遅延時間。
東京から東京 →低レイテンシー 東京から海外 →高レイテンシー
SLA（Service Level Agreement）（エスエルエー）
一言でいうと サービス品質を保証する契約。
稼働率99.99% など。
列指向データベース（Column-oriented Database）（カラム オリエンテッド データベース）
一言でいうと
データを「行」ではなく「列」ごとに保存するデータベースです。

向いている用途
  • 売上分析

  • 銀行システム

代表例
  • Amazon Redshift
  • Google BigQuery
  • Snowflake
NoSQLとの違い NoSQLは 「データ構造」の考え方。 列指向DBは 「データの保存方法」の考え方。 全く別の分類です。
Amazon EMR（Amazon Elastic MapReduce）（イーエムアール）
一言でいうと HadoopやSparkを簡単に使えるサービス。
数億件のアクセスログ分析 売上分析 ビッグデータ分析
Apache Hadoop（アパッチ ハドゥープ）
大量データを複数台のコンピューターで分散処理する仕組み。
Apache Spark（アパッチ スパーク）
Hadoopより高速なデータ分析エンジン。
最近はこちらが使われることも多いです。
EC2インスタンス（Elastic Compute Cloud）（イーシーツー）
一言でいうと AWS上の仮想サーバー。
Windows Server Ubuntu など好きなOSを入れられます。
リザーブドインスタンス
長期間使う前提で予約し、 料金を安くする仕組み。
3年間使うことが決まっている 社内システム向け。
スポットインスタンス
空いているサーバーを格安で借りる仕組み。
AI学習 動画変換 大量計算 など途中で止まっても困らない処理。
AWSの都合で停止されることがあります。 その代わり非常に安価です。
Amazon Athena（アマゾン アシーナ）
一言でいうと S3上のファイルをSQLで検索できるサービス。
CSV Parquet JSON などをSQLで分析できます。
gz（gzip）（ジーゼット）
一言でいうと データ圧縮形式。
100MB ↓ 20MB のように小さくできます。
Athenaでは 圧縮したCSV(gz)を読み込むこともできます。 容量が減るため、 保存コストや読み込み時間の削減につながります。
Amazon S3（Amazon Simple Storage Service）（エススリー）
一言でいうと
AWS上にファイルを保存するためのサービスです。

画像や動画、PDF、CSV、バックアップなど、 さまざまなファイルを保存できます。 「クラウド上の巨大なハードディスク」 と考えるとイメージしやすいです。

保存できるもの
  • 画像
  • 動画
  • PDF
  • CSV
  • ログファイル
  • バックアップ
  • ホームページのHTML
S3の構造 
S3

└ Bucket（バケット）
      ├ logo.png
      ├ report.pdf
      ├ movie.mp4
      └ sales.csv
「Bucket（バケット）」は ファイルを保存する箱です。 Windowsのフォルダーに近いイメージですが、 S3では最初にバケットを作り、その中にファイルを保存します。
使用例
  • Webサイトの画像保存
  • バックアップ保存先
  • ログ保存
  • Athenaで分析するCSV保存
  • 動画配信

講師メモ S3は「データベース」ではありません。 ファイルを保存する場所です。 社員一覧を保存するならRDSやDynamoDB、 画像やPDFを保存するならS3、 という使い分けになります。
