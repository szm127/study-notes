Azure 基礎まとめ

AzureはMicrosoftが提供するクラウドサービスです。
Windowsとの相性が非常によく、企業システムでよく利用されています。 

Azure
│
├─ サブスクリプション（契約）
│
├─ 仮想ネットワーク（VNet）
│     └─ サブネット
│
├─ 仮想マシン
│      ├─ Windows Server
│      └─ Ubuntu/CentOS
│
└─ Active Directory
Windows Server（ウィンドウズ サーバー）
一言でいうと
サーバー向けのWindowsです。

普段使うWindows 11は個人向けですが、 Windows Serverは企業でサーバーとして使うためのOSです。

使用例
  • ファイルサーバー
  • 社内Webサーバー
  • Active Directoryサーバー
  • データベースサーバー
例え
Windows11＝家庭用車 Windows Server＝トラック どちらも車ですが、 用途が違います。
Hyper-V（ハイパーブイ）
一言でいうと Windowsで仮想マシンを動かすためのハイパーバイザー(制御プログラム)。

1台のPCの中に 複数のOSを動かせます。

Windows ↓ Hyper-V ↓ Windows Server Ubuntu CentOS などを同時に動かせます。
AWSにもAzureにも仮想マシンがありますが、 Hyper-Vは 「その土台」 になっている技術です。
CentOS（セントオーエス）
一言でいうと サーバーでよく使われてきたLinuxです。
Webサーバー データベース メールサーバー など。
最近はCentOSの開発方針が変わったため、 Rocky LinuxやAlmaLinuxへ移行する企業も増えています。 「CentOSというLinuxがある」 くらい知っていれば十分です。
Ubuntu（ウブントゥ）
一言でいうと 初心者にも人気のLinux。
AWS EC2 Azure VM GCP Compute Engine などでも非常によく使われます。
研修や開発環境では Ubuntuが登場することが多いです。
Linux（リナックス）
一言でいうと
無料で利用できる代表的なOS（オペレーティングシステム）の一つです。

OS（オペレーティングシステム）とは、 パソコンやサーバーを動かすための基本ソフトウェアです。 私たちが普段使うWindowsやmacOSと同じ仲間で、 Linuxは特にサーバーやクラウド環境で広く利用されています。

代表的なOS
  • Windows
  • macOS
  • Linux
  • Android（Linuxをベースに開発）
Linuxにはさまざまな種類（ディストリビューション）がある 
Linux
├ Ubuntu
├ CentOS
├ Rocky Linux
├ AlmaLinux
├ Debian
└ Red Hat Enterprise Linux（RHEL）
「Linux」というOSが1つあるのではなく、 Linuxをもとに作られたさまざまな種類があります。 Windowsでいうと、 「Windows 10」「Windows 11」のような違いではなく、 同じ土台から作られた別の製品というイメージです。
クラウドでの利用例
  • AWS EC2でUbuntuを起動する
  • Azure VMでRocky Linuxを利用する
  • Google CloudでDebianを利用する
企業のクラウド環境では、 Windows ServerよりLinuxが選ばれることも多くあります。
Linuxがサーバーでよく使われる理由
  • 無料で利用できるものが多い
  • 動作が軽い
  • 安定性が高く、長時間稼働に向いている
  • カスタマイズしやすい
  • クラウドサービスとの相性が良い
Windowsとの違い
項目 Windows Linux
主な用途 個人PC・企業PC サーバー・クラウド
操作方法 GUI（画面操作）が中心 CUI（コマンド操作）が中心
費用 ライセンスが必要 無料のものが多い
※Linuxにも画面操作（GUI）はありますが、サーバーではコマンド操作がよく使われます。
例え
OSを「自動車」とすると、
  • Windows：街乗り向けの乗用車
  • Linux：荷物をたくさん運べるトラック
どちらも車ですが、得意な用途が違います。
講師メモ
受講者から 「UbuntuとLinuxって違うんですか？」 と聞かれたら、 「UbuntuはLinuxの一種です。」 と答えれば十分です。 また、 「CentOSやRocky LinuxもLinuxの一種です。」 まで説明できると理解が深まります。
ネットワーク・仮想ネットワーク・サブネット
ネットワーク コンピューター同士が通信できる仕組み。
会社のLAN 家庭のWi-Fi インターネット すべてネットワークです。
仮想ネットワーク（Virtual Network：VNet）（バーチャル ネットワーク） Azure上に作る自分専用のネットワーク。
AWSのVPCとほぼ同じ役割です。
サブネット（Subnet）（サブネット） 仮想ネットワークを区画分けしたもの。
覚え方 ネットワーク ↓ VNet（土地） ↓ サブネット（土地の区画） ↓ 仮想マシン（家）
Active Directory（アクティブ ディレクトリ）
一言でいうと 会社のユーザーやPCをまとめて管理する仕組み。
社員1000人いる会社で ・ログインID ・パスワード ・所属部署 ・利用できるPC を一括管理できます。
例え 学校の職員室。 先生全員の情報を管理していて、 誰がどの教室を使えるか などを決めています。
Azureにおけるサブスクリプション（Subscription）（サブスクリプション）
一言でいうと Azureを利用するための契約・料金管理の単位。
会社では ・開発用 ・本番用 ・検証用 でサブスクリプションを分けることがあります。
Netflixのサブスクとは少し意味が違います。 Azureでは 「利用料金をまとめる箱」 というイメージです。 その箱の中に 仮想マシン データベース ストレージ などを作ります。
